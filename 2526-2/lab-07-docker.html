1.
Tổng quan Docker & Cài đặt

Giới thiệu Docker

Định nghĩa
Docker là một nền tảng mã nguồn mở cho phép đóng gói ứng dụng và các thành phần phụ thuộc vào các container, giúp ứng dụng chạy ổn định trên nhiều môi trường khác nhau.
  • Dễ dàng triển khai và mở rộng ứng dụng.
  • Cung cấp không gian làm việc gọi là containers.
  • Giải quyết triệt để lỗi: "Code chạy được trên máy tôi nhưng lỗi ở máy khác".

Tại sao cần Docker?

🌟 Lợi ích Cốt lõi

  • ✅ Môi trường chuẩn: Docker Image giúp chuẩn hóa môi trường.
  • 🛠️ Khắc phục triệt để: Lỗi kinh điển "chạy được trên máy tôi" nhờ tái tạo môi trường dễ dàng.
  • 📦 Tối ưu chi phí: Build 1 lần → Chạy ở mọi nơi (Dev/Test/Prod).
  • 🚀 Triển khai siêu tốc: Chỉ cần một lệnh docker run.

🤖 Tối ưu cho Hệ thống AI

  • 🧠 Đóng gói phức tạp: Đóng gói gọn gàng toàn bộ môi trường AI (Python, PyTorch, CUDA, …).
  • 🔄 Reproducibility: Đảm bảo tính lặp lại (đồng nhất) hoàn hảo cho quá trình training & inference trên nhiều server.
  • 🎮 Tận dụng GPU: Hỗ trợ phần cứng cực tốt thông qua NVIDIA Container Toolkit.

Docker vs Virtual Machine

Kiến trúc Docker

  • Docker Client: Cung cấp giao diện dòng lệnh (CLI) để người dùng tương tác thông qua các lệnh (docker run, docker build).
  • Docker Daemon: Thành phần chạy ngầm trên Docker Host, chịu trách nhiệm quản lý containers, images, networks và volumes.
  • Docker Registry: Nơi lưu trữ và phân phối Docker Image. Docker Hub là registry công cộng lớn nhất.
💡 Hiểu đơn giản: Docker Client gửi lệnh → Docker Daemon nhận và thực thi → Kéo Image từ Registry về để chạy.

Kiến trúc Docker

Cài đặt Docker

  • Windows: Sử dụng Docker Desktop.
  • Linux: Cài đặt Docker Engine thông qua trình quản lý gói như apt (Ubuntu), yum (CentOS, RHEL).
  • macOS: Sử dụng Docker Desktop.
  • Tải bộ cài đặt tại: https://docker.com/products/docker-desktop

Kiểm tra cài đặt

Mở Terminal / Command Prompt và gõ các lệnh sau:


                docker --version
                docker run hello-world
                docker ps -a

Lệnh docker run hello-world sẽ tải image từ Docker Hub và chạy container đầu tiên.

2.
Docker Image & Container

Image

Định nghĩa
Docker Image là bản mẫu (template) chỉ đọc, chứa mọi thứ để chạy ứng dụng: OS nền, Thư viện, Mã nguồn, Cấu hình.
  • Bất biến (Immutable): Không thay đổi sau khi được tạo.
  • Tầng lớp (Layered): Cấu tạo từ nhiều lớp giúp tối ưu lưu trữ và tái sử dụng.
  • 💡 Quan hệ: Giống OOP, Image (Class) là khuôn mẫu để sinh ra các Container (Object).

🖼️ Docker Image & Cấu trúc Layer

💾 Tiết kiệm: Chia sẻ layer chung.
⚡ Build nhanh: Nhờ Layer Cache.
🔄 Tái sử dụng: Dùng chung Base Image.
🚀 Push/Pull: Chỉ tải layer thiếu.
⏪ Rollback: Quay về bản cũ tức thì.
🛠️ Bảo trì: Quản lý theo từng tầng.
💡 Image (Class) ➜ Container (Object)

Cấu trúc Layered tối ưu hóa lưu trữ và truyền tải dữ liệu

Khái niệm Container

Định nghĩa
Docker Container là một instance (phiên bản đang chạy) được sinh ra từ Docker Image.

🛡️ Môi trường

Cách ly ở mức tiến trình (process-level) nhưng vẫn chia sẻ chung hệ điều hành (Kernel) với máy Host.

⚠️ Lưu ý Dữ liệu

Container có thể ghi file tạm, nhưng dữ liệu này sẽ mất hoàn toàn khi bị xóa.

→ Cần dùng Volume để lưu trữ vĩnh viễn.

Dockerfile cơ bản

Định nghĩa: File văn bản (kịch bản) chứa các chỉ thị để Docker tự động đóng gói Image.
  • FROM: Kế thừa Base Image (OS/Runtime).
  • WORKDIR: Thiết lập thư mục làm việc trong container.
  • COPY: Copy mã nguồn từ máy Host vào Image.
  • RUN: Chạy lệnh cài đặt thư viện/phụ thuộc.
  • CMD: Lệnh mặc định thực thi khi Container chạy.
  • ENV, EXPOSE, ENTRYPOINT,...
Dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY . .
RUN pip install torch numpy
CMD ["python", "train.py"]
💡 Bản chất: Mỗi dòng lệnh trong Dockerfile sẽ tạo ra 1 layer mới.
→ Lệnh ít thay đổi (FROM, RUN pip) nên để ở trên cùng để tận dụng Cache!
🚀 Lệnh build image:
Cú pháp:
$ docker build -t <tên_image>:<tag> <đường_dẫn>
Ví dụ:
$ docker build -t my-app:latest .

Docker Hub & Lệnh cơ bản

☁️ Docker Hub: Nền tảng Registry đám mây công cộng lớn nhất để lưu trữ và chia sẻ Image.
🏷️ Versioning: Quản lý các phiên bản Image qua tag (Ví dụ: python:3.10, nginx:latest).
bash - Vòng đời Container
$ docker pull <image_name>:<tag> # Tải image từ Docker Hub
$ docker run -d -p 8080:80 <image_name> # Khởi chạy thành container
$ docker ps # Xem danh sách container đang chạy
$ docker stop <container_id> # Dừng container
$ docker rm <container_id> # Xóa container

Chia sẻ Image lên Docker Hub

💡 Kịch bản: Bạn vừa build xong một Image và muốn chia sẻ cho nhóm. Hãy đẩy (Push) nó lên Docker Hub để mọi người có thể dễ dàng tải về!
bash - Publish Image
# 1. Đăng nhập vào tài khoản Docker Hub của bạn
$ docker login
# 2. Gắn thẻ (Tag) đúng chuẩn: <username>/<repo>:<tag>
$ docker tag my-app:latest yourusername/my-app:1.0
# 3. Đẩy Image lên Docker Hub
$ docker push yourusername/my-app:1.0
🤝 Thực hành: Sau khi Push thành công, bạn hãy gửi tên Image (ví dụ: yourusername/my-app:1.0) cho người bạn bên cạnh. Mời bạn mình dùng lệnh docker pull để tải về và chạy thử trên máy của họ

3.
Docker Storage & Network

Docker Storage

⚠️ Vấn đề:
Dữ liệu được tạo ra và ghi bên trong Container sẽ bị xóa sạch hoàn toàn khi Container đó bị dừng và xóa bỏ.
(Vì lớp "writable layer" của Container chỉ mang tính chất tạm thời).
💡 Giải pháp:
Sử dụng Docker Storage để gắn một không gian lưu trữ từ bên ngoài vào. Giúp dữ liệu tồn tại độc lập với vòng đời của Container. Container có chết, dữ liệu vẫn an toàn!

Shared Storage

💡 Cơ chế chia sẻ: Một Docker Volume có thể được gắn (mount) vào nhiều container cùng lúc. Mọi thay đổi dữ liệu từ một container sẽ ngay lập tức xuất hiện ở các container còn lại.
(Ví dụ: Container Nginx đọc dữ liệu từ Volume, trong khi Container App ghi dữ liệu vào cùng Volume đó)

Các cơ chế lưu trữ

📦 Volume
  • Do Docker quản lý hoàn toàn (nằm trong vùng an toàn của Docker).
  • Độc lập với cấu trúc file của máy Host (Portable).
  • Dùng cho: Dữ liệu lâu dài, quan trọng (Database, Logs...).
🔗 Bind Mount
  • Do Host quản lý (Ánh xạ trực tiếp một thư mục từ máy Host vào Container).
  • Phụ thuộc vào đường dẫn thực tế trên Host.
  • Dùng cho: Phát triển phần mềm (Sửa code trên Host, Container nhận ngay).
⚡ Tmpfs Mount
  • Lưu trữ trực tiếp trên RAM của máy Host.
  • Không ghi ra ổ cứng, dữ liệu tự mất khi Container dừng.
  • Dùng cho: Dữ liệu tạm thời, thông tin nhạy cảm (Keys, Passwords) hoặc cần tốc độ đọc/ghi cực nhanh.

Các cơ chế lưu trữ

Các lệnh cơ bản với Docker Storage

📦 1. Quản lý Docker Volume

# Tạo volume mới
$ docker volume create my-vol
# Liệt kê & Kiểm tra
$ docker volume ls
$ docker volume inspect my-vol
# Xóa volume
$ docker volume rm my-vol

🚀 2. Mount Volume vào Container

# Sử dụng flag -v hoặc --mount
$ docker run -d -v my-vol:/data nginx
💡 Dữ liệu trong /data sẽ được lưu vào my-vol và không mất khi container bị xóa.

🔗 3. Bind Mount (Host ↔ Container)

# Mount thư mục hiện tại vào nginx
$ docker run -d -v $(pwd)/html:/usr/share/nginx/html nginx
💡 Phù hợp để dev: Sửa file ở máy host, web cập nhật ngay lập tức.

⚡ 4. Tmpfs Mount (RAM only)

# Mount vào bộ nhớ tạm RAM
$ docker run -d --tmpfs /tmp nginx
💡 Dữ liệu cực nhanh nhưng sẽ mất hoàn toàn khi container dừng.

Network & Port Mapping

🌐 Docker Network
(Giao tiếp nội bộ)
  • Tạo mạng nội bộ để các container giao tiếp với nhau.
  • 💡 DNS nội bộ: Các container cùng mạng gọi nhau trực tiếp bằng Tên Container.
  • → Không cần Port Mapping khi giao tiếp nội bộ.
# Ví dụ: App gọi DB qua tên 'db-server'
$ docker run --name db-server --network my-net ...
🚪 Port Mapping
(Truy cập từ bên ngoài)
  • Cho phép truy cập dịch vụ bên trong container từ máy Host hoặc Browser.
  • Cú pháp: -p HostPort:ContainerPort
  • Mở "cổng" để thế giới bên ngoài nhìn thấy container.
# Truy cập Web qua cổng 8080 máy Host
$ docker run -p 8080:80 nginx
🚀 Tóm lại: Network dùng cho giao tiếp "nội bộ" giữa các container, còn Port Mapping dùng để "mở cửa" cho bên ngoài truy cập.

Các loại Network & Lệnh quản trị

🛠️ Các Network Driver phổ biến
• Bridge (Mặc định):

Tạo mạng nội bộ ảo để các Container giao tiếp với nhau. Đây là loại được dùng nhiều nhất.

• Host:

Container dùng chung network stack với máy Host (không có sự cô lập mạng, hiệu năng cao nhất).

• None:

Container hoàn toàn không có kết nối mạng (cô lập tuyệt đối).

🚀 Lệnh cốt lõi
# Tạo mạng mới
$ docker network create my-net
# Chạy container vào mạng
$ docker run --network my-net ...
⚙️ Quản trị hệ thống
# Liệt kê các mạng
$ docker network ls
# Xóa một mạng
$ docker network rm my-net
* Lưu ý: Lệnh docker network prune dùng để dọn dẹp các mạng không sử dụng (cẩn trọng khi dùng).

"Mỗi thành phần là một Container riêng biệt..."

😫 "Khó khăn khi chạy thủ công
  • Gõ đi gõ lại hàng chục lệnh docker run.
  • Khó thiết lập mạng (Network) cho các Container gọi nhau.

💡 Giải pháp: Một file duy nhất - Một lệnh duy nhất!

Cấu trúc file Docker Compose

🏷️ version: Phiên bản Docker Compose (vd: 3.9).
📦 services: Danh sách các Container cần chạy.
🖼️ build/image: Dùng Image có sẵn hoặc tự Build.
🔌 ports: Mở cổng (Máy_thật:Container).
💾 volumes: Lưu dữ liệu vĩnh viễn (ổ đĩa).
⏳ depends_on: Thứ tự chạy (Cái nào chạy trước).
🌐 networks: Tạo mạng nội bộ giữa các Container.
📄 docker-compose.yml YAML v3.9 
version: "3.9"
services:
    frontend:
    image: ai-system-frontend:latest
    build: ./frontend
    ports:
        - "8080:80"
    networks:
        - app_network
    depends_on:
        - backend
    backend:
    image: ai-system-backend:latest
    build: ./backend
    ports:
        - "5000:5000"
    volumes:
        - ./backend:/app
    networks:
        - app_network
networks:
    app_network:
    driver: host

Vận hành hệ thống Compose

🚀 Triển khai (Up)

Build image và khởi chạy toàn bộ container ở chế độ nền (detached):

$ docker compose up -d
🛑 Dừng & Xóa (Down)

Dừng container và xóa bỏ các tài nguyên mạng, volume tạm thời:

$ docker compose down -v
💡 Tip: Dùng docker compose logs -f để theo dõi nhật ký hoạt động của tất cả service.

Thực hành 1: Docker Compose

Bài 1: Frontend + Model Service
  • Viết file docker-compose.yml chạy 2 service: frontend và model-service.
  • frontend hiển thị giao diện đơn giản cho người dùng.
  • model-service trả về kết quả dự đoán giả lập dưới dạng JSON.
  • Mục tiêu: luyện chạy nhiều service và kết nối giữa các service bằng tên service.
Bài 2: Model Service + Database
  • Viết file docker-compose.yml chạy 2 service: model-service và database.
  • model-service lưu đầu vào hoặc kết quả dự đoán vào database.
  • Cấu hình volume để dữ liệu database không bị mất khi container bị xóa.
  • Mục tiêu: luyện volume, service name và quản lý hệ thống AI nhiều thành phần.
Chạy hệ thống bằng: docker compose up -d
Dọn dẹp bằng: docker compose down -v
